Service access endpoints#

The Rackspace Ticketing API has two environments with global endpoints.

Perform all testing and development against the demo environment. This prevents test tickets from negatively impacting your support experience.

Demo environment#

https://demo.ticketing.api.rackspace.com

Production environment#

https://ticketing.api.rackspace.com

The ticketing event feed also has two environments available for use. Events triggered within the demo environment are published to the demo feed, while production events are published to the primary ticketing event feed.

Demo feed#

https://lon.feeds.api.rackspacecloud.com/consolidated_ticketing/events/<accountID>

Production feed#

https://ord.feeds.api.rackspacecloud.com/consolidated_ticketing/events/<accountID>
