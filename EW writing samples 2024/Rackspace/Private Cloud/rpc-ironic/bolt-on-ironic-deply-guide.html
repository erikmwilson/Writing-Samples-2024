Ironic Bolt-On Installation Guide#

Pre-Requisites#

This assumes that RPC-O Newton is already configured and installed.

r14.3.0 was tested and works

Ironic Setup#

  1. Add entries into /etc/openstack_deploy/conf.d/ironic.yml that satisfy your specific environment. Example shown below.

    ironic-compute_hosts:
  infra01:
    ip: <infra01_management_ip>
  infra02:
    ip: <infra02_management_ip>
  infra03:
    ip: <infra03_management_ip>
ironic-infra_hosts:
  infra01:
    ip: <infra01_management_ip>
  infra02:
    ip: <infra02_management_ip>
  infra03:
    ip: <infra02_management_ip>

  2. Follow pre-install steps to add the variables specific for ironic.

  3. Re-run /opt/rpc-openstack/openstack-ansible/scripts/dynamic-inventory.py in order to update the inventory with the Ironic and Swift inventory items.

  4. After this is done, run the playbooks from /opt/rpc-openstack/openstack-ansible/playbooks/

    openstack-ansible setup-hosts.yml --skip-tags=V-38660
openstack-ansible os-neutron-install.yml
openstack-ansible haproxy-install.yml
openstack-ansible os-swift-install.yml
openstack-ansible os-ironic-install.yml

Set up and configure the neutron network#

  1. Ensure you have fully deployed OpenStack before continuing.

    Note

    As of the Ocata release, the neutron CLI is deprecated. Refer to the openstack-pythonclient documentation for more up-to-date commands.

    neutron net-create --shared --provider:physical_network tftp --provider:network_type flat tftp
neutron subnet-create --name ironic-tftp \
--allocation-pool start=<NON_ALLOCATED_TFTP_RANGE_START>,end=<NON_ALLOCATED_TFTP_RANGE_END> \
--dns-nameserver=8.8.4.4 tftp <TFTP_NETWORK>/<TFTP_NETMASK>

  2. Execute the following command:

    neutron net-list | grep tftp | awk '{print $2}'

  3. Take the return UUID from the above command, and add the content to the user_osa_variables_overrides.yml:

    ironic_ironic_conf_overrides:
  neutron:
    cleaning_network_uuid: "<NEUTRON_TFTP_NETWORK_UUID>"
  conductor:
    automated_clean: true

    This automatically cleans the nodes on check-in.

Re-run os-ironic-install.yml.

From here, you can use ironic post install to perform the post installation steps.
